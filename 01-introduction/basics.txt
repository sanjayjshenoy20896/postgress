Non Relational Databases:
Organise data into one or more tables
1.) each table has columns and rows
2.) A unique key identifies each row.

eg: mySql,Oracle,PostreSQL,mariaDb

Tables and keys:
Data are arranged in rows and columns
The columns name defines the keys in the table
A row defines the values correspoding to column name. Each row defines a record.
Each row has a unique called primary key. This attribute is used to uniquely define a record in db.
